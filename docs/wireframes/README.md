# Wireframes

Thư mục này lưu wireframe theo từng màn hình chính của SoccMan. Mỗi wireframe nên thể hiện bố cục desktop, tablet/mobile nếu có khác biệt đáng kể, trạng thái loading/error/empty và các vùng tương tác chính.

## Danh sách wireframe cần có

- [x] **Dashboard** (`dashboard.md` hoặc file thiết kế tương ứng)
  - Tổng quan KPI bán hàng, công việc, báo giá, đơn hàng và cảnh báo cần xử lý.
- [x] **Customer List** (`customer-list.md` hoặc file thiết kế tương ứng)
  - Danh sách khách hàng, bộ lọc, tìm kiếm, phân trang và hành động nhanh.
- [x] **Customer Detail** (`customer-detail.md` hoặc file thiết kế tương ứng)
  - Hồ sơ khách hàng, timeline tương tác, báo giá/đơn hàng liên quan và tác vụ theo dõi.
- [x] **Quotation** (`quotation.md` hoặc file thiết kế tương ứng)
  - Màn hình tạo/xem báo giá, bảng dòng sản phẩm, tổng tiền, trạng thái phê duyệt và gửi khách.
- [x] **Order Detail** (`order-detail.md` hoặc file thiết kế tương ứng)
  - Thông tin đơn hàng, trạng thái xử lý, thanh toán, giao hàng và lịch sử cập nhật.
- [x] **Task Board** (`task-board.md` hoặc file thiết kế tương ứng)
  - Bảng Kanban theo trạng thái, thẻ công việc, bộ lọc người phụ trách và thao tác kéo-thả.

## Quy ước đặt tên

- Dùng chữ thường và dấu gạch ngang cho tên file, ví dụ `customer-detail.md`.
- Nếu wireframe được lưu bằng công cụ bên ngoài, thêm link và ảnh chụp phiên bản hiện tại vào file Markdown tương ứng.
- Ghi rõ ngày cập nhật, người phụ trách và phạm vi thay đổi ở cuối mỗi file wireframe.
